<br>

 **Análisis de archivos .js. Proceso enfocado en revisar, mapear y comprender el código JavaScript de una aplicación para identificar vectores de ataque potenciales. Incluye la detección de funciones sensibles, endpoints ocultos, parámetros interesantes, lógica de validación, fugas de datos y posibles superficies para XSS, DOM Clobbering, prototipo pollution o bypass de seguridad.<br><br>
Objetivo: convertir cada archivo .js en una fuente de inteligencia útil durante la fase de recon y explotación, revelando rutas y comportamientos que no están expuestos directamente en la interfaz de la aplicación.**

• Pentest view (opcional): heurísticas para resaltar señales (CORS permisivo, cookies débiles, TRACE expuesto, WebDAV abierto, fingerprinting de servidor), sin tocar tu JSON.</a>.**

<br><br>

Herramienta fuera de línea en el navegador para gestionar conocimiento de HTTP: métodos y headers con estados y descripciones. Podés crear/editar/eliminar, cargar en masa, contar por categoría, buscar globalmente y exportar/importar JSON.<br><br>
• Formato: HTML/JS estático.<br>
• Datos: en memoria + Export/Import JSON (y localStorage si activás el parche de persistencia).<br><br>

<br><br>

- Categorías dinámicas

Secciones y categorías
• Métodos core (GET, HEAD, POST, PUT, DELETE, CONNECT, OPTIONS, TRACE) con estado CORE.
• Extensiones estandarizadas (PATCH, WebDAV/DeltaV/CalDAV, SEARCH, etc.) con fuente/RFC.
• No estándar “de la vida real” (p. ej. PURGE, BAN) con campo “Dónde se ve”.
• Headers (≈100+ y ampliable): estándar, de facto, obsoletos, seguridad, CORS, CH, H2/H3, proxies, tracing, WebSocket, WebDAV, históricos.
• Cada categoría muestra contador automático de ítems.<br><br><br>

- CRUD por categoría
• Agregar ítem (pide nombre + descripción + campos extra según categoría).
• Eliminar último (revierte el último agregado manualmente).
• Alta masiva (nombres): pegás una lista (uno por línea) → crea ítems “pendientes”.
• Completar masivo (descripciones): pegás descripciones por orden o Nombre: descripción para “match”.
• Dedupe recomendado (case-insensitive, normaliza -/_).<br><br><br>

- Buscador global
• Panel lateral fijo.
• Filtra por nombre, descripción, estado y categoría, en todas las tablas.<br><br><br>

- Exportar / Importar
• Exportar JSON: incluye meta (fecha, autor, versión), métodos (core/ext/no estándar) y headers.
• Importar JSON: restaura el estado completo (ideal para compartir packs en tu repo).
• Schema sugerido: meta.version, meta.schema = "aquiles-http/v1" para compat futura.<br><br><br>

- Persistencia local (opcional)
• Parche opcional para guardar/recuperar desde localStorage (incluye término de búsqueda).
• Sigue funcionando 100% offline. Si no activás el parche, todo vive en memoria y se conserva vía JSON.<br><br><br>

- Interfaz
• Tema oscuro con glassmorphism.
• Encabezados de tabla “sticky” al hacer scroll.
• Panel lateral con buscador + export/import.
• Píldoras de estado: CORE / EXTENSIÓN / NO ESTÁNDAR; en headers: Sí / No / De facto / Obsoleto / Borrador / Custom.<br>

<br><br>


1. Abrí el archivo .html en tu navegador.

2. Usá las secciones (Core / Extensión / No estándar / Headers).

3. Agregar o Alta masiva para cargar nuevos métodos/headers.

4. Completar masivo para descripciones (por orden o Nombre: descripción).

5. Filtrá con el buscador o navegá por categorías con contadores.

6. Exportar JSON para versionar/compartir; Importar JSON para cargar paquetes.

7. (Opcional) Activá el parche de localStorage si querés persistencia automática entre sesiones.
